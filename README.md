# ОПЕРАЦИОННАЯ ДОСКА

**ОПЕРАЦИОННАЯ ДОСКА** — локальное desktop-приложение для macOS для личного управления задачами руководителя по отделам.

Приложение работает **полностью офлайн**, без сервера, без Docker и без авторизации. Все данные сохраняются в локальный JSON-файл.

---

## 1) Что это за приложение

Это личный тасктрекер операционного директора в минималистичном стиле:
- канбан + список задач;
- быстрые фильтры, поиск, сортировка;
- управление отделами, исполнителями, статусами;
- локальное автосохранение.

---

## 2) Что умеет

### Основное
- Два режима: **Канбан** и **Список**.
- Drag-and-drop задач между статусами.
- Автообновление статуса/завершения при переносе в «Готово».
- Полностью русскоязычный интерфейс.
- Локальная дата в формате `дд.мм.гггг`.

### Работа с задачами
- Создание, редактирование, удаление задачи.
- Поля задачи:
  - название
  - описание
  - отдел
  - исполнитель
  - приоритет
  - статус
  - дата начала
  - дедлайн
  - цвет
  - флаг «Срочно»

### Фильтры и аналитика
- Фильтры по статусу, отделу, исполнителю, приоритету, дедлайну.
- Поиск по названию.
- Скрытие завершённых.
- Подсветка просроченных.
- Дашборд: активные, просроченные, на проверке, критические.
- Блок «Что горит сейчас».

### Справочники
- Отделы: создание, редактирование, цвет, порядок, архив.
- Исполнители: создание, редактирование, инициалы, цвет, порядок, архив.
- Статусы: создание, редактирование, порядок, архив.

## Быстрый запуск без установки (через index.html)

Теперь приложение можно открыть сразу после скачивания проекта:

1. Скачайте/распакуйте репозиторий.
2. Откройте файл `index.html` двойным кликом в браузере.
3. Все задачи сохраняются автоматически в `localStorage` браузера и восстанавливаются при следующем открытии `index.html`.

> Важно: данные хранятся локально в том же браузере и профиле пользователя.

---

## 3) Как установить зависимости

> Требуется: **Node.js 20+** и **npm**.

```bash
npm install
```

---

## 4) Как запустить локально (режим разработки)

```bash
npm run dev
```

Что произойдет:
1. Поднимется Vite (frontend).
2. Запустится Electron-окно desktop-приложения.
3. Все изменения в коде будут подхватываться автоматически.

---

## 5) Как собрать приложение для Mac

### Сборка кода
```bash
npm run build
```

### Сборка `.dmg`
```bash
npm run dist:mac
```

Готовый артефакт будет в папке `dist/` (и папках electron-builder).

---

## 6) Где хранятся данные

Приложение сохраняет данные в JSON-файл в `userData` Electron:

- файл: `oper-board-data.json`
- путь зависит от macOS-пользователя (показывается в интерфейсе в карточке «Хранилище»)

Все изменения сохраняются автоматически (autosave).

---

## 7) Как сделать резервную копию JSON вручную

1. Откройте приложение.
2. Посмотрите карточку **«Хранилище»** на главном экране.
3. Найдите файл `oper-board-data.json`.
4. Скопируйте его в безопасное место (например, iCloud Drive/внешний диск).

Для восстановления:
- закрыть приложение,
- заменить текущий `oper-board-data.json` на резервный,
- снова открыть приложение.

## 8) Структура проекта

```text
.
├─ electron/
│  ├─ main.ts          # окно Electron + локальное JSON-хранилище
│  ├─ preload.ts       # безопасный мост API в renderer
│  └─ tsconfig.json
├─ src/
│  ├─ lib/
│  │  ├─ date.ts       # форматирование дат, дедлайны
│  │  └─ task.ts       # сортировка и цвета приоритетов
│  ├─ App.tsx          # основной интерфейс приложения
│  ├─ main.tsx         # вход React
│  ├─ styles.css       # Tailwind + базовые UI-классы
│  ├─ types.ts         # типы доменной модели
│  └─ vite-env.d.ts
├─ README.md
└─ package.json
```

---

## 9) Технологии

- **Electron** — desktop shell для macOS.
- **React + TypeScript** — UI и типобезопасность.
- **Vite** — быстрая сборка и dev-среда.
- **Tailwind CSS** — минималистичный UI.
- **JSON file storage** — простое локальное хранение без БД.

---

## 10) Что входит в MVP

- Главная доска + дашборд
- Канбан-представление
- Список задач
- Экран справочников (отделы, исполнители, статусы)
- Просмотр и редактирование задачи
- Локальное автосохранение в JSON
- Полностью офлайн-режим
